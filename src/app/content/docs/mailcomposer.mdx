## Overview

MailComposer explores how AI can simplify email template creation by transforming user input into polished email copy and production-ready HTML templates. The project provides a streamlined user experience with intuitive UI feedback that generates downloadable email packages with minimal effort.

## Goals

<ul className="list">
<li>Generate email content from prompts</li>
<li>Instant preview and download generated HTML output</li>
<li>Validation and error notifications upon form input</li>
<li>Prevent accidental multiple submissions by cooldown period and rate limiting</li>
</ul>

## Architecture

<ul className="list">
<li>UI pattern and component reusability</li>
<li>API route and page views</li>
<li>Type-safe form data handling and AI prompts</li>
<li>State management with React Hook Form</li>
<li>Error handling and form input validation</li>
<li>Deployment readiness and production verification</li>
</ul>

## Features

**1. AI Email Composition**

Generate structured email drafts from user prompts using a streamlined composition flow.

**2. HTML Preview**

Preview rendered email HTML directly in the browser using an iframe-based preview page.

**3. HTML Export**

Download generated HTML in an extractable Zip file for use in email clients or external workflows.

**4. Toast Notifications**

Responsive toast feedback powered by Sonner for:

<ul className="list">
<li>Validation errors</li>
<li>API failures</li>
<li>Success states</li>
</ul>

**5. Cooldown + Rate Limiting**

Implemented request protection using Redis-based rate limiting to help prevent abuse and accidental spam requests.

**6. TypeScript + ESLint Verification**

Project includes strict linting and verification workflows to reduce deployment issues and improve maintainability.

## Challenges & Learning

<ul className="list">
<li>Designing reusable utility functions while balancing DRY principles with readability and intent</li>
<li>Separating business logic and presentation responsibilities for clearer architecture and easier theming</li>
<li>Utilizing comprehensive form schemas, HTML5 validation, type guards, and input restrictions</li>
<li>Structuring form input into thoughtful AI prompts for more reliable output generation</li>
<li>Rendering form fields and template themes dynamically through conditional UI patterns</li>
<li>Extracting reusable MJML/HTML generation logic into shared utilities, allowing preview and export functionality to rely on the same source of truth</li>
<li>Evaluating architectural trade-offs, including when suppressing a one-off linter warning was the more practical decision</li>
<li>Building a cohesive error messaging system to create a smoother user experience</li>
</ul>
