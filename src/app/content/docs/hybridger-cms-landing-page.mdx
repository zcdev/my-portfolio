## Overview

HyBridger was built to show how a professional-grade, production-ready web architecture comes to life using `Next.js`, `Contentful`, `TailwindCSS`, and `TypeScript`. The project demonstrates real-world content modeling, `GraphQL` querying patterns, image optimization, and scalable UI composition, offering a practical example of how engineering and system design integrate in modern workflows.

## Goals

<ul className="list">
<li>Build a high-performance headless CMS landing page</li>
<li>Demonstrate `GraphQL` querying and type-safe data handling</li>
<li>Develop a reusable component system for sections and UI elements</li>
<li>Implement a fast, predictable content pipeline (`Contentful` → `Next.js` → `Vercel`)</li>
<li>Create a scalable foundation that can support future expansion (blog, chatbot, and additional sections)</li>
</ul>

## Architecture
HyBridger uses the `Next.js App Router` with a well-organized structure:

<ul className="list">
<li>Server components for headless CMS data fetching</li>
<li>Client components for interactive UI pieces</li>
<li>Centralized `GraphQL` utilities</li>
<li>Custom mobile-responsive image component</li>
<li>Stongly typed data models for every query and CMS field</li>
<li>ISR revalidation through a custom API route</li>
</ul>
This ensures predictable data flow and fast rendering time.

## Challenges & Solutions

**CVE-2025-55182 — React Server Component Vulnerability**

During deployment, an industry-wide RSC exploit caused a failure in Vercel’s build pipeline.
The fix required upgrading:

<ul className="list">
<li>`Next.js`</li>
<li>`React` server rendering dependencies</li>
<li>Contentful’s SDK (to ensure compatibility)</li>
</ul>

Outcome: A valuable real-world experience in diagnosing and resolving dependency-level security issues in a production environment.

**GraphQL Type Management**

Challenge: Managing multiple queries and fields became complex as the project grew.

Solution: Consolidated all types and queries under a dedicated lib directory.

**Image Variant Complexity**

Challenge: Ensuring pixel-perfect image display across dark mode, mobile, and desktop.

Solution: Introduced a flexible prop-driven image component supporting multiple variants.
