## Chat Flow Logic

The core behavior is driven by:

**1. Message sequencing**
```
setMessages(prev => [...prev, { from: "user", text: userText }]);
```
**2. Step progression**
```
setStep(prev => prev + 1);
```
**3. UX: Thinking delay**

Displayed using a controlled timeout:
```
setTimeout(() => setIsThinking(false), 2500);
```
**4. Fix for double-input issues**

Originally, `setStep()` was inside the timeout, causing race conditions when the user clicked rapidly.

Solution: update step synchronously, not after a delay.

## Challenges & Solutions

**Managing State Across Multiple Components**

Challenge: Passing message state, step progression, and callbacks required careful planning to avoid prop drilling and maintain clarity.

Solution: breaking down each component's responsibility and clearly identifying the needs of associated TypeScript definition to improve organization and data flow orchestration.

**Sequencing & Timing Bugs**

Challenge: Delay inside state setters caused chat flow inconsistencies.

Solution: reorganize the event loop and thinking cycle to isolate UI delay from logic.
